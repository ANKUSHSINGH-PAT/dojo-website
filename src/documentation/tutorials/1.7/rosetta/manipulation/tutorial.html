$ to dojo.query

Querying in Dojo is accomplished via the dojo.query API. There is an excellent tutorial on dojo.query which I suggest you read at some point for more background. Let's take a quick look at using dojo.query. 

//Lookup all "p" tags using jQuery
$("p");

//Lookup all "p" tags using dojo.query
dojo.query("p");

Simple, but not very useful, after all, we want to actually do something with the nodes we look up.

Let's .addClass

First in the list of Manipulation APIs is .addClass which as you may have guessed adds a CSS class to an element. 

//.addClass using jQuery
$("p").addClass("selected");

//.addClass using Dojo
dojo.query("p").addClass("selected");

.after that?

To duplicate jQuery's .after API we have to first import Dojo's NodeList-manipulate extensions. This is because .after is not built in to dojo.query by default. Dojo uses dojo.require to add in functionality without having to add in separate script tags. No need to specify a path, Dojo figures this out automatically. Once you've required the code you don't need to require it again. Let's take a look at implementing .after 

//.after using jQuery
$('.inner').after('<p>Test</p>');

//.after using Dojo
dojo.require("dojo.NodeList-manipulate");
dojo.query(".inner").after('<p>Test</p>');

In both cases all nodes with the .inner class name are queried and a p tag with the text "Test" is inserted after each found .inner node.

.append-ing

There are two major functions in jQuery for appending nodes, .append and .appendTo. There are multiple ways to do this with Dojo. Here are the API mapped append and appendTo functions. 

//.append using jQuery
$('.inner').append('<p>Test</p>');

//.append using Dojo
dojo.require("dojo.NodeList-manipulate");
dojo.query(".inner").append('<p>Test</p>');

Select an element on the page and append it to another

//.appendTo using jQuery
$("h2").appendTo($(".container"));

//.appendTo using Dojo
dojo.require("dojo.NodeList-manipulate");
dojo.query("h2").appendTo(".container");
